## Phase 1: 问题理解

### 1.1 赛题类型识别

执行步骤：
1. 读取赛题全文
2. 提取关键词
3. 匹配问题类型

**类型判定规则**：

| 关键词 | 类型 | 典型模型 |
|--------|------|---------|
| 预测、趋势、未来 | 预测 | ARIMA, LSTM, 灰色预测 |
| 评价、排序、优选 | 评价 | TOPSIS, AHP, 熵权法, DEA |
| 最优、最小、最大 | 优化 | LP, NLP, GA, PSO, 蒙特卡罗优化 |
| 路径、覆盖、规划 | 路径 | 贪心, Dijkstra, 动态规划 |
| 决策、策略、博弈 | 决策 | DP, MDP, 博弈论 |
| 仿真、模拟、随机 | 仿真 | 蒙特卡罗, 排队仿真, 元胞自动机 |
| 分类、聚类、识别 | 分类 | SVM, 神经网络, K-means |
| 图像、遥感、医学影像 | 图像 | 边缘检测, 分割, 特征提取 |
| 效率、投入产出 | 效率 | DEA 数据包络分析 |
| 降维、主成分、因子 | 降维 | PCA, 因子分析 |

### 1.2 约束识别

从赛题中提取：
- **硬约束**：必须满足（如时间限制、容量上限）
- **软约束**：尽量满足（如重叠率10%-20%）
- **隐含约束**：从问题背景推导（如物理定律、常识）

**论文实例 (B311)**：
- 硬约束：测线必须覆盖整个海域
- 软约束：重叠率10%-20%，测线总长度最短
- 隐含约束：测量船航行平稳，海平面恒定

### 1.3 目标明确

- 单目标：直接优化
- 多目标：判断主次关系
  - 有明显主次 → 主要目标法（将次要目标转为约束）
  - 无明显主次 → 加权法 / 帕累托优化

## Phase 2: 模型选择

### 2.1 决策树选择法

```
问题特征
├── 几何/空间关系?
│   ├── 二维 → 三角函数+正弦定理
│   ├── 三维 → 空间几何+向量分析
│   └── 曲面 → 微分几何+数值方法
├── 时间序列?
│   ├── 线性+平稳 → ARIMA
│   ├── 非线性 → LSTM/GRU
│   └── 小样本 → GM(1,1)
├── 优化?
│   ├── 线性约束 → LP/MILP
│   ├── 非线性 → NLP/启发式算法
│   └── 多目标 → 主要目标法/加权法
├── 决策?
│   ├── 确定型 → 动态规划
│   ├── 随机型 → MDP
│   └── 多人 → 博弈论
└── 数据驱动?
    ├── 有标签 → 监督学习
    └── 无标签 → 聚类/降维
```

### 2.2 模型组合策略

**递进式**：由简到繁，逐步深入
```
基础模型 → 扩展模型 → 高级模型
例: 动态规划 → MDP → 博弈论 (B175)
```

**互补式**：各取所长，相互验证
```
解析模型(建模) + 数值模型(求解) + 可视化(验证)
例: 几何分析 + Matlab计算 + CAD验证 (B311)
```

**流水线式**：前一步输出作为后一步输入
```
数据预处理 → 核心建模 → 优化求解 → 结果验证
例: 随机森林拟合 → 几何建模 → 飞蛾火焰算法 (B477)
```

### 2.3 模型论证要求

选择模型后必须说明：
1. **为什么选这个模型**：问题特征与模型特性的匹配
2. **为什么不选其他模型**：对比分析
3. **模型的假设条件**：明确适用范围
4. **模型的预期效果**：理论上的优势

## Phase 3: 模型构建

### 3.1 假设建立

- 简化问题但不改变本质
- 每条假设必须有理由
- 禁止无理由的假设、与问题矛盾的假设、过度简化导致模型失真

### 3.2 符号定义

- 表格形式：符号 | 说明 | 单位
- 与正文一致，首次出现时定义
- 同一变量全文一致，区分大小写

### 3.3 公式推导

1. 每步有依据（定理/定义/已知条件）
2. 编号连续
3. 关键步骤有文字说明
4. 最终公式有物理解释

### 3.4 代码实现

- 变量名与公式符号一致
- 有注释说明对应公式编号
- 有输入输出说明
- 模块化设计

## Phase 4: 求解验证

### 4.1 求解策略

| 策略 | 适用场景 | 工具 |
|------|---------|------|
| 解析求解 | 公式可直接求解 | 手推+符号计算 |
| 数值求解 | 公式复杂 | Matlab/Python |
| 智能优化 | 大规模/非凸 | `metaheuristic` (GA/PSO/SA/ACO) |
| 仿真求解 | 随机过程 | `monte_carlo` (积分/排队/随机游走) |
| 综合评价 | 多指标排序 | `evaluation` (TOPSIS/DEA/PCA) |
| 图像分析 | 遥感/医学图像 | `image_processing` (检测/分割/特征) |

### 4.2 验证方法

**解析验证**：极限情况检验、量纲分析、边界条件检验

**数值验证**：与题目数据对比、与其他方法交叉验证、敏感性分析

**可视化验证**：CAD/几何画板作图、直观检查结果合理性

### 4.3 敏感性分析

1. 确定关键参数
2. 设定变化范围
3. 逐个改变参数
4. 记录输出变化
5. 绘制敏感性曲线
6. 确定关键因素
