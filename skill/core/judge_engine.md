# Judge Engine v3.0

> 数学建模论文评审引擎。融合教材方法论 + 获奖论文经验。
>
> 更新：新增模型合理性、参数解释、实验可信度、结果可迁移性评审维度。

---

## 评分体系（总分100分）

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 模型质量 | 30分 | 建模逻辑、模型合理性、公式推导、参数解释 |
| 验证完整性 | 25分 | 求解过程、验证方法、敏感性分析、实验可信度 |
| 创新性 | 20分 | 方法创新、视角创新、应用创新 |
| 表达质量 | 15分 | 结构清晰、术语统一、结果解释深度、可迁移性 |
| 工程质量 | 10分 | 代码规范、可复现性、可视化质量 |

---

## 各维度评分细则

### 1. 模型质量 (30分)

| 分数段 | 标准 |
|--------|------|
| 27-30 | 假设合理且有理由，符号完整，推导严谨，模型选择有论证，参数有物理解释 |
| 22-26 | 假设基本合理，符号基本完整，推导有过程，模型选择有说明 |
| 16-21 | 假设不够充分，符号有遗漏，推导不够详细，参数解释不足 |
| 0-15 | 假设不合理，符号混乱，公式无推导，模型与问题不匹配 |

**新增检查项**：
- 模型合理性：模型是否与问题本质匹配？
- 参数解释：每个参数是否有物理解释或实际意义？
- 模型选择论证：是否说明为什么选这个模型而不是其他？

### 2. 验证完整性 (25分)

| 分数段 | 标准 |
|--------|------|
| 23-25 | 三级验证体系完整（解析+数值+仿真），敏感性分析充分，实验可信度高 |
| 18-22 | 有多种验证方法，有基本敏感性分析 |
| 12-17 | 有验证但不充分，敏感性分析不足 |
| 0-11 | 无验证，或验证方法与问题不匹配 |

**三级验证要求**：

| 级别 | 方法 | 要求 |
|------|------|------|
| Level 1 | 解析验证 | 极限情况检验、量纲分析、边界条件检验 |
| Level 2 | 数值验证 | 与已知数据对比、交叉验证、残差分析 |
| Level 3 | 仿真验证 | 蒙特卡洛模拟、多次运行取统计量 |

**敏感性分析要求**：
- 至少改变2个关键参数
- 绘制敏感性曲线
- 确定关键因素
- 分析参数变化对结果的影响程度

### 3. 创新性 (20分)

| 分数段 | 标准 |
|--------|------|
| 18-20 | 提出新的建模方法或改进现有方法，有明确创新点论述 |
| 14-17 | 方法组合有新意，有创新意识但不够突出 |
| 10-13 | 使用常规方法，但应用得当 |
| 0-9 | 方法与问题不匹配，无任何创新 |

### 4. 表达质量 (15分)

| 分数段 | 标准 |
|--------|------|
| 14-15 | 结构清晰，术语统一，每个结果有分析，有一般性策略，有局限性讨论，结论可迁移 |
| 11-13 | 结构基本清晰，术语基本统一，结果有分析 |
| 7-10 | 结构有改进空间，术语不统一，讨论不充分 |
| 0-6 | 结构混乱，表达不清，无结果分析 |

**新增检查项**：
- 结果解释深度：每个数值结果是否有物理解释？
- 可迁移性：结论是否可以推广到其他类似问题？
- 一般性策略：是否从具体结果中提炼了通用规律？

### 5. 工程质量 (10分)

| 分数段 | 标准 |
|--------|------|
| 9-10 | 代码规范（变量名与公式一致），可复现，可视化风格统一 |
| 7-8 | 代码基本规范，可复现 |
| 5-6 | 代码可运行但不够规范 |
| 0-4 | 无代码，或代码不可运行 |

## 常见缺陷清单

### 高频缺陷（扣分5-10分）
1. 模型选择无理由
2. 公式无推导过程
3. 结果无验证
4. 无敏感性分析
5. 结果无分析
6. 模型与问题不匹配

### 中频缺陷（扣分2-5分）
1. 假设无理由
2. 符号不完整
3. 图表不规范
4. 术语不统一
5. 讨论不充分
6. 参数无解释

### 低频缺陷（扣分1-2分）
1. 错别字
2. 编号错误
3. 格式不统一
4. 代码变量名与公式不一致

## 自动修正流程

```
评审 → 评分 → score < 90?
├── 是 → 定位缺陷 → 生成修改建议 → 执行修正 → 重新评审
│       (最多循环3轮)
└── 否 → 输出最终论文
```

### 修正优先级

1. **模型质量缺陷**（影响最大，30分权重）→ 补充推导、选择论证、假设说明、参数解释
2. **验证完整性缺陷**（25分权重）→ 补充三级验证、敏感性分析、提高实验可信度
3. **创新性缺陷**（20分权重）→ 明确创新点、方法改进论述
4. **表达质量缺陷**（15分权重）→ 补充结果解释、一般性策略、局限性讨论
5. **工程/图表缺陷**（10分权重）→ 规范代码、统一术语、修正格式

## 使用方法

```
/mm-review adversarial fix
```

1. 输入论文全文
2. 引擎逐维度评分
3. 输出评分报告
4. 如 score<90，自动修正
5. 最多循环3轮
6. 输出最终论文+评分报告
