# Paper Generator v2.0

> 数学建模论文生成引擎。基于竞赛评审标准，生成结构化、高质量的论文。

---

## 论文结构模板

```
摘要 (300-500字)
一、问题重述
  1.1 问题背景
  1.2 需要求解的问题
二、问题分析
  2.1 问题一的分析
  2.2 问题二的分析
  ...
三、模型假设
四、符号说明
五、模型的建立与求解
  5.1 问题一模型的建立与求解
  5.2 问题二模型的建立与求解
  ...
六、模型的评价与推广
  6.1 模型的优点
  6.2 模型的缺点
  6.3 模型的推广
参考文献
附录
```

## 各部分写作规范

### 摘要

**结构**：问题→方法→结果→价值

**模板**（三选一，详见 `patterns/abstract_patterns.md`）：

- **递进式**：多问递进型赛题
- **框架式**：整体性较强的赛题
- **方法导向式**：方法创新性强的论文

**关键规则**：
1. 数值必须具体："覆盖率达99.7%"而非"覆盖率很高"
2. 方法必须命名："基于动态规划的多阶段决策模型"
3. 每问独立成句
4. 禁止空泛表述

### 问题重述

**1.1 问题背景**：简述背景（3-5句），引出核心问题

**1.2 需要求解的问题**：逐条列出，保持原文表述，编号清晰

### 问题分析

**模板**（每个问题）：
```
针对问题[X]，[分析问题本质]。
首先[第一步思路]，然后[第二步思路]，最后[第三步思路]。
```

**关键规则**：分析思路而非求解过程；体现逻辑推理链；说明为什么选择这种方法。

### 模型假设

- 编号列表 + 理由
- 简化问题但不改变本质
- 禁止无理由的假设

### 符号说明

- 表格形式：符号 | 说明 | 单位
- 按出现顺序排列
- 首次出现时定义

### 模型的建立与求解

**每个问题的结构**：
```
5.X 问题X模型的建立与求解
5.X.1 模型的建立
  [问题背景/情境描述]
  [图X: 示意图]
  [公式推导过程]
5.X.2 模型的求解
  [代入数据计算]
  [表X: 计算结果]
  [结果分析]
```

**关键规则**：先建模后求解；公式有推导过程；每个公式有编号；结果有表格和分析。

### 模型的评价与推广

- 优点：方法创新点、结果精度、适用范围广
- 缺点：假设的局限性、计算复杂度、未考虑的因素
- 推广：可以应用到哪些场景、需要做哪些修改

## 术语统一规范

| 标准术语 | 禁止使用 |
|---------|---------|
| 覆盖宽度 | 覆盖范围、测量宽度 |
| 重叠率 | 重复率、重叠比例 |
| 测线 | 航线、测量线 |
| 动态规划 | DP（正文首次出现用全称） |
| 马尔可夫决策过程 | MDP（正文首次出现用全称） |

## 禁止事项

1. **流水账**：禁止"然后...然后...然后..."
2. **只有公式没有解释**：每个公式后必须有文字说明
3. **无推导过程**：关键公式必须有推导
4. **无结果分析**：每个结果后必须有分析
5. **无模型对比**：必须说明为什么选择这个模型
6. **无验证**：必须有某种形式的验证
7. **空泛结论**：禁止"取得了良好效果"
8. **无图表**：关键结果必须有图表支撑
