# 评估与验证规则库 (Evaluation Rules)

## 验证方法体系

### Level 1: 解析验证
- 用已知条件代入公式，检查是否得到预期结果
- 极限情况检验（如坡度=0时退化为平坦情况）
- 量纲分析（公式两边单位一致）

### Level 2: 数值验证
- 与题目给定数据对比
- 与其他方法结果交叉验证
- 敏感性分析（改变输入参数观察输出变化）

### Level 3: 仿真验证
- 蒙特卡洛模拟
- 多次运行取统计量
- 与理论值对比

---

## 评估指标设计原则

| 指标类型 | 具体指标 | 适用场景 |
|---------|---------|---------|
| 精度 | 误差率、RMSE、R² | 预测类 |
| 效率 | 时间复杂度、计算时间 | 算法类 |
| 覆盖 | 覆盖率、漏测率 | 规划类 |
| 收益 | 最终资金、收益率 | 决策类 |
| 稳定性 | 敏感性系数 | 所有类型 |

---

## 敏感性分析模式

### 单因素敏感性
```
固定其他参数 → 改变一个参数 → 观察输出变化 → 绘制敏感性曲线
```

### 多因素敏感性
```
正交试验设计 → 多参数组合 → 方差分析 → 确定关键因素
```

---

## 结果表达规范

1. **数值结果**：保留适当精度（一般2-4位有效数字）
2. **表格**：对齐、有单位、有表头说明
3. **图形**：有标题、坐标轴标签、图例、单位
4. **结论**：每个结果后跟一句解释/分析
