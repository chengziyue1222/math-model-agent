## 2. 阶段 1：选题与分析

**目标**：理解题意，拆解子问题，确定建模方向。

### 命令
```
/mm-analyze
```

### 示例对话
```
> /mm-analyze
请分析以下题目：
【2022 年国赛 A 题】波浪能最大输出功率问题。
设计一个振荡浮子式波浪能转换装置，在给定波浪条件下，
通过优化浮子半径、吃水深度、弹簧刚度和阻尼系数，
最大化平均输出功率。
```

### 产出
- `analysis.md` — 问题类型识别（优化问题）、子问题拆解（运动建模→能量转换→参数优化）、方法候选列表、时间分配建议

### 下一步
→ `/mm-literature` 或 `/nature-search`

- 基于 `analysis.md`，搜索经典方法
- 输出：多维对比表（精度/难度/创新性/可解释性）→ `literature.md`

→ `/mm-model`

## 4. 阶段 3：建模与求解

### 4.1 构建模型
```
/mm-model
```
- 输入：`analysis.md` + `literature.md`
- 产出：`model.md`（假设列表、符号表、目标函数、约束条件、求解算法伪代码）

### 4.2 数据预处理（如有原始数据）
```
/data-analysis ./math-model/data/wave_data.csv
```
- 5 阶段流水线：加载→清洗→EDA→统计检验→特征工程
- 产出：`cleaned_data.csv`、分析脚本、分析报告

### 4.3 数值求解
```
/mm-solve
```
- 输入：`model.md` + 数据文件
- 使用 L-BFGS-B 优化、多起点搜索、灵敏度分析
- 产出：`code/solve.py`、`results.json`

### 求解过程示意
```python
# solve.py 核心逻辑
from scipy.optimize import minimize

def objective(params):
    r, d, k, c = params
    # 模型方程：浮子运动 + 能量转换
    P = compute_average_power(r, d, k, c)
    return -P  # 最大化 → 最小化负值

result = minimize(objective, x0, method='L-BFGS-B', bounds=bounds)
# 最优解：r=5.0m, d=10.0m, k=204226.5 N/m, c=100000 N·s/m
# 最大平均功率：3205.51 W
```

→ `/mm-visualize`

## 4.1 扩展建模技能（v5.0 新增）

### 竞赛选题与路线选择 🆕
```
/mm-route
```
- CUMCM A/B/C 三题对比：问题卡（领域/数学核心/风险/拥挤度/团队适配）
- 方法路线卡（15+字段）：最小可行路线、强力论文路线、雄心路线
- 工程可行性门控（7问）、反同质化规则（14条）
- 输出：12节结构化选题报告
- 参考：`~/ai-skills/math-skills/references/selection-rubric.md`

### 通用建模工作流 🆕
```
/mm-workflow
```
- 5阶段竞赛工作流：理解→架构设计→实现→验证调试→结果分析
- 成本函数模式：时变速度、U型能耗、载荷依赖、软时间窗、碳定价
- 求解策略：贪心构造 + 2-opt局部搜索 + 车队管理
- 交付物：求解器脚本(.py)、结果(.json/.txt)、图表(.png)

### 数据建模与可视化 🆕
```
/mm-data
```
- 检查点驱动：读数据→[检查点1：确认目标]→EDA→[检查点2：确认模型]→建模→出图
- 自动检测变量类型，推荐模型（回归/分类/时序/探索）
- 样本量<30降级为描述性分析，>=50必须划分训练/测试集
- 出版级可视化：OO API、色盲友好调色板、强制标签

### 运筹优化建模 🆕
```
/mm-or
```
- 6步流程：问题理解(6维)→文献搜索→逆向验证→模型构建→自检→LaTeX输出
- 约束模块速查 C1-C12：容量/时间窗/唯一分配/流平衡/互斥/条件激活等
- 19类运筹问题LaTeX公式库：LP/IP/TSP/VRP/排队论/库存/博弈论/机制设计
- 求解策略：小规模(<1000)→精确算法，中规模→分支定界，大规模→元启发式

## 13. 统一建模流程

> 所有模型兼容的11步标准化流程。
>
> 详见 `skill/modeling_pipeline.md`

```
问题理解 → 假设建立 → 变量定义 → 目标明确 → 约束识别
    ↓
模型选择 → 求解计算 → 结果验证 → 敏感性分析 → 结果解释 → 论文输出
```

**每步要求**：
1. 问题理解：提取关键词，识别问题类型
2. 假设建立：每条假设有理由
3. 变量定义：符号|说明|单位 表格
4. 目标明确：单目标/多目标有主次/多目标有优先级
5. 约束识别：硬约束/软约束/隐含约束
6. 模型选择：说明选择理由（参考模型决策引擎）
7. 求解计算：代码变量名与公式一致
8. 结果验证：三级验证（解析/数值/仿真）
9. 敏感性分析：至少改变2个关键参数
10. 结果解释：四层讨论（结果→对比→策略→局限）
11. 论文输出：先建模后求解，公式有推导

## 14. 论文生成引擎

> 基于竞赛评审标准的结构化论文生成系统。
>
> 详见 `skill/paper_generator.md`

### 摘要模板（三选一）

**递进式**（多问递进型赛题）:
```
针对问题1，运用[方法]，求解[目标]，得到[结果]。
针对问题2，在问题1基础上[推广/改进]，引入[新方法]，得到[结果]。
...
关键词：[3-6个]
```

**框架式**（整体性较强的赛题）:
```
本文对[背景]进行研究。首先[基础模型]；针对[扩展]建立[扩展模型]；
在[复杂情形]时，引入[新思想]，建立[高级模型]。
...
```

**方法导向式**（方法创新性强的论文）:
```
针对[问题]，本文基于[核心思想]建立了[模型体系]。
[方法1]用于[场景1]，[方法2]用于[场景2]...
```

### 论文结构

```
摘要 (300-500字)
一、问题重述（背景+问题列表）
二、问题分析（每问独立分析思路）
三、模型假设（编号+理由）
四、符号说明（符号|说明|单位）
五、模型的建立与求解（先建模后求解，公式有推导）
六、模型的评价与推广（优点+缺点+推广）
参考文献
附录（代码+详细数据）
```

### 禁止事项

- 禁止流水账（"然后...然后..."）
- 禁止只有公式没有解释
- 禁止无推导过程
- 禁止无结果分析
- 禁止空泛结论（"取得了良好效果"）

1. 逐维度评分

3. 生成修改建议
